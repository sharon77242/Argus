### 0.2.3 — `src/licensing/clock-guard.ts` (Enterprise tier only)

```ts
// Monotonic clock delta check — zero network calls, in-container only
const agentStartWallMs = Date.now();
const agentStartHrns = process.hrtime.bigint();

export function checkClockIntegrity(tier: string, nowMs: number): 'ok' | 'rollback' {
  if (tier !== 'enterprise') return 'ok'; // Self-Hosted Pro: accepted trade-off
  const elapsedNs = process.hrtime.bigint() - agentStartHrns;
  const expectedWallMs = agentStartWallMs + Number(elapsedNs / 1_000_000n);
  if (expectedWallMs - nowMs > 60_000) return 'rollback'; // 60s NTP tolerance
  return 'ok';
}
```

### 0.3.1 — Correlation IDs (`src/instrumentation/correlation.ts`)

**Why first:** When the demo GIF shows "N+1 detected", it currently shows the query in isolation. With correlation, it shows "47 queries during GET /users/preferences — N+1 detected". That is a 10x more compelling demo.

Uses `AsyncLocalStorage` to propagate a request context through the entire async call chain.

```ts
import { AsyncLocalStorage } from 'node:async_hooks';

export interface RequestContext {
  requestId:  string;
  method?:    string;
  url?:       string;
  startedAt:  number;
}

const storage = new AsyncLocalStorage<RequestContext>();

export function runWithContext<T>(ctx: RequestContext, fn: () => T): T {
  return storage.run(ctx, fn);
}

export function getCurrentContext(): RequestContext | undefined {
  return storage.getStore();
}
```

Wire into `HttpTracer` — wrap the incoming request handler in `runWithContext`. All events emitted during that request automatically include `requestId`, `method`, `url`.

Update all event payloads to include `correlationId?: string` from `getCurrentContext()?.requestId`.

Update `ProfilerEvent`, `QueryEvent`, `HttpEvent`, `LogEvent` types in `src/instrumentation/types.ts`.

**Tests:** `tests/instrumentation/correlation.test.ts`
- [ ] Context available inside async callbacks nested 5 levels deep
- [ ] Context is isolated between concurrent requests (two parallel `runWithContext` calls don't bleed)
- [ ] `getCurrentContext()` returns undefined outside a request context
- [ ] QueryEvent includes correlationId when emitted inside runWithContext
- [ ] QueryEvent correlationId is undefined when emitted outside context

### 0.3.2 — Graceful Shutdown helper (`src/profiling/graceful-shutdown.ts`)

```ts
export class GracefulShutdown {
  private registered = false;

  register(agent: DiagnosticAgent, opts: { timeoutMs?: number } = {}): this {
    if (this.registered) return this;
    this.registered = true;
    const timeout = opts.timeoutMs ?? 5000;

    const shutdown = (signal: string) => {
      agent.emit('info', `DiagnosticAgent: ${signal} received — flushing telemetry`);
      const timer = setTimeout(() => {
        agent.emit('info', 'DiagnosticAgent: flush timeout — exiting');
        process.exit(0);
      }, timeout);
      timer.unref(); // don't hold the process open if flush completes first
      agent.stop();
      process.exit(0);
    };

    process.once('SIGTERM', () => shutdown('SIGTERM'));
    process.once('SIGINT',  () => shutdown('SIGINT'));
    return this;
  }
}
```

Wire into builder as `.withGracefulShutdown(opts?)` — returns `this` for chaining.

**Tests:** `tests/profiling/graceful-shutdown.test.ts`
- [ ] Registers SIGTERM and SIGINT handlers
- [ ] Calling register() twice is idempotent (no duplicate handlers)
- [ ] agent.stop() is called on signal
- [ ] Timeout fires if agent.stop() hangs

### 0.3.4 — Slow Require Detector (`src/profiling/slow-require-detector.ts`)

Measures module load times at startup. Fires once per module, synchronously.

```ts
import { createRequire } from 'node:module';

export class SlowRequireDetector {
  private thresholdMs: number;
  private timings: Map<string, number> = new Map();

  constructor(opts: { thresholdMs?: number } = {}) {
    this.thresholdMs = opts.thresholdMs ?? 100;
  }

  patch(emitter: EventEmitter): this {
    // Hook Module._load (Node internals) or use --require with performance marks
    // Preferred: diagnostics_channel 'module.cjs.load' available in Node 20+
    return this;
  }

  getSlowModules(): Array<{ module: string; durationMs: number }> {
    return [...this.timings.entries()]
      .filter(([, ms]) => ms >= this.thresholdMs)
      .map(([module, durationMs]) => ({ module, durationMs }))
      .sort((a, b) => b.durationMs - a.durationMs);
```

## 0.8 — Self Code Review

Issues found and corrected before shipping:

| Issue | File | Fix |
|---|---|---|
| `agentStartWallMs` and `agentStartHrns` must be module-level singletons | `src/licensing/clock-guard.ts` | Move both to module scope — not inside the check function. Re-initializing each call defeats the purpose. |
