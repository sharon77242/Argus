# Phase 5 (Future): AI-Powered Fix Generation

## Overview
A separate **out-of-container** service that consumes the OTLP telemetry exported by the Deep Diagnostic Agent and uses an LLM to generate concrete code fix diffs.

> **Why separate?** LLM API calls are expensive, latency-heavy, and unpredictable. They must never run inside a production container under load.

## Architecture

```
┌──────────────────────┐       OTLP/mTLS       ┌──────────────────────────┐
│  Production Container │  ──────────────────▶  │  AI Fix Generation Svc   │
│  (Diagnostic Agent)   │                       │                          │
│  - anomalies          │                       │  1. Receives anomalies   │
│  - fix suggestions    │                       │  2. Fetches source code  │
│  - static scan results│                       │  3. Sends to LLM        │
└──────────────────────┘                       │  4. Returns unified diff │
                                                │  5. Opens PR / Issue     │
                                                └──────────────────────────┘
```

## Proposed Components

### 1. Telemetry Ingestion Server
- OTLP HTTP receiver (Node.js + Express or Fastify)
- mTLS client cert verification (mirrors the agent's exporter config)
- Stores anomalies in a lightweight queue (Redis/BullMQ or in-memory)

### 2. Source Code Resolver
- Given a `sourceLine` from an anomaly (e.g. `src/orders.ts:42:8`), fetch the surrounding code context (±20 lines)
- Supports: local filesystem, GitHub API, GitLab API
- Respects `.gitignore` and never sends `node_modules` or secrets

### 3. LLM Fix Generator
- Constructs a structured prompt from:
  - The anomaly type (slow query, memory leak, event loop block)
  - The fix suggestions already generated by Level 1
  - The surrounding source code context
  - The static analysis warnings from Level 2
- Sends to OpenAI / Anthropic / local Ollama endpoint
- Parses the LLM response into a unified diff format

### 4. Fix Delivery
- **GitHub Integration:** Opens a PR with the suggested diff
- **Slack/Teams:** Posts a formatted message with the fix
- **Dashboard:** Displays in a web UI alongside the telemetry data
- **CLI:** `diagnostic-agent fix --anomaly-id <id>` fetches and applies locally

## Security Constraints
- Source code sent to the LLM must go through the same entropy scrubbing pipeline
- No database values, secrets, or environment variables in prompts
- Rate-limit LLM calls to prevent cost runaway (max N fixes per hour)
- All fix diffs must go through CI/CD — no auto-apply to production

## Dependencies
- Requires Level 1 (Fix Suggestions) and Level 2 (Static Analysis) to be implemented first
- Requires an LLM API key or local model endpoint
- Optional: GitHub/GitLab API token for PR creation
